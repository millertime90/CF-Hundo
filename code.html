The code for HUNDO

                    
                        function displayHundo() { 

                            let isEven_orOdd = function(n) { 
                        
                                if(n == 0) { return "odd"; } 
                                else if (n % 2 == 0) { return "even"; } 
                                else { return "odd"; }
                            
                            }; 
                        
                            let startValue = document.querySelector("#startValue"); 
                            let endValue = document.querySelector("#endValue"); 
                            let results = document.querySelector("#results"); 
                        
                            results.innerHTML = ""; 
                        
                            for(let i = Number.parseInt(startValue.value); i <= Number.parseInt(endValue.value); i++) { 
                        
                                results.append((() => {
                        
                                    let tdata = document.createElement("TR"); 
                                    tdata.innerHTML = `<td class=${isEven_orOdd(i)}>${i}</td>`; 
                        
                                    return tdata; 
                        
                                })()); 
                        
                            } 
                        
                        }

The code is structured in one function.

displayHundo

displayHundo, when invoked, is the program that runs to display a range of numbers based on user input in the input element fields displayed in "The App" page of this website in accordance to the challenge of this project: to display 0 and odd numbers with regular font, and even numbers with bold font. The entire block of code scoped in the function consists of four declared and initialized, local variables, functionality to reset the Hundo, and a sub block of code scoped within a for loop which runs per loop as per the expressions of the for loop.

The four variables are as follows in the order in which they are declared:

  • isEven_orOdd
  • startValue
  • endValue
  • results

isEven_orOdd

isEven_orOdd is declared and initialized as a function to return either of two string values, "even", or "odd". These values serve to assign value to the class attributes of the table-row HTML elements to be appended by the HTML element node, tbody#results. As this function suggests, these string values are class names which are to reference toward CSS rules each respective table-row HTML element's table-data elements are to inherit. These value returns are determined by the boolean evaluations stated by the if, else if, and else statements. These evaluations compare a passed argument, n, equivalency to 0 (returning "odd" if true) in the if statement and n % 2 equivalency to 0 (returning "even" if true) in the else if statement. The else statement executes a default value return, "odd", granted both the if and else if statement evaluations return false. These evaluations are executed consecutively in that order and stop executing granted a true returned evaluation.

startValue & endValue

startValue & endValue are declared and initialized variables that reference the user input field elements. It's also important to note that while they have a type attribute assigned a string value of "number", both of these variable's value property are of a string data type. Observing the for loop that follows, these value properties are parsed into numerical data types so that they can be compared as numbers instead of strings so as to avoid a lexicological comparison which would result in a semantic error (i.e. "2" < "100" === false).

results

results is a declared and initialized variable that references a div HTML element node which is to contain and display the range of numbers returned by the Hundo.


The code that follows . . .

The code that follows when executed: results.innerHTML = "" resets the Hundo upon new user input by accessing results' innerHTML property and assigning it an empty string value ("").

The for loop that follows puts the value properties of startValue & endValue to use. In the initial expression, a variable, i, is declared and initialized to startValue's value property parsed into an integer by it being passed as an argument into the Number.parseInt method. As this is the initial expression of a for loop, this expression is executed only once. Next, the condition expression executes. It compares a less than or equivalancy comparison from i to Number.parseInt(endValue.value), i, being the "Start Value" (numerical parsed) input by the user and Number.parseInt(endValue.value) being the numerical parsed "End Value" input by the user. So as long as this evaluation returns true, the code block scoped by the for loop will execute per loop consequentially following after the condition expression, then lastly i++ executes. This order of operations will continue to execute until the condition expression returns false.

The for loop's code block

The code block within the for loop when executed calls upon results's append method. The argument that's passed through this method is of a node type returned by an ES6 IIFE which executes a block of code itself declaring tdata, initializing it to an HTML element node, table-row element by calling the docuemnt's createElement method with passed argument "TR", accessing then tdata's innerHTML property and assigning it a string value utilizing ES6 functionality (${} syntax) to access local/global variables. In this case, it's to stringify either of the aforementioned "even" or "odd" values returned by isEven_orOdd with i passed in as an argument and within this string assigning `${i}` as the innerHTML of what will be rendered as a table-data HTML element as such:
`<td class=${isEven_orOdd(i)}>${i}</td>`

Lastly, tdata is returned thus appending it to the innerHTML of results.
